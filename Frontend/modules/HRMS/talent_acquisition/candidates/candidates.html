🔄 Architecture: How Data Flows
1 · Google Form → Google Sheet
Applicants submit the form. Responses auto-populate Sheet1 columns A–L (Timestamp through Source).
2 · Candidate Dashboard → Live Sheet
This page calls the Apps Script doGet() endpoint to read all rows. Hit 🔄 Refresh anytime to pull the latest data.
3 · HR Actions → Write back to Sheet
Shortlisting, stage changes, and HR notes call doPost(updateCandidate). Column N = shortlisted, O = stage, P = notes. Changes persist in the Sheet.
4 · Shortlisted → Interview Panel
Shortlisted candidates are saved to localStorage and automatically surfaced in the Interview Panel for scheduling rounds.
⚙️ Set your Apps Script deployment URL in SHEET_SCRIPT_URL at the top of this file's <script> block. Deploy the script as "Anyone" with execute access.
